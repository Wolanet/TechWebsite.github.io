## Hard rules — read first

1. **CSS edits go in `assets/css/theme.css` ONLY.** Never touch `main.css` or any SASS file *for styling changes* — `theme.css` is loaded after `main.css` and overrides it. (One documented exception, update#42: two lines of **dead cruft** were deleted directly from `main.css` with the user's explicit sign-off — the unused Google-Fonts `@import` (Merriweather/Source Sans Pro) and a reference to the already-deleted `bg46.jpg`. That was dead-code removal, not styling. All *styling* still goes in theme.css only.)
2. **Cache-bust on every CSS/JS change.** All HTML files carry `main.css?v=N`, `theme.css?v=N` and `site.js?v=N`. Bump N across all HTML files (Git Bash `sed`) whenever theme.css OR site.js changes, or fixes will not appear on the live site.
3. **Use Git Bash (`sed`, `grep`) for any string substitution in HTML/CSS.** PowerShell emits curly quotes (U+201C/201D) into files, breaking CSS class selectors.
4. **Commit message format: `website update#N` — nothing else.** No body, no Co-Authored-By, no bullet points, no links.
5. **Current CSS cache version: `?v=33`** (bumped in update#50 — self-hosted fonts + hero entrance-fade removal + smooth scroll). Next CSS/JS change → bump to `?v=34`.
6. **Next commit number: `update#54`.**
7. **`generic.html` is a REDIRECT STUB, not a real page** (recreated in update#52) — the original was deleted in update#9 but stayed in Google's index (titled just "Tek Tsunami", it was surfacing as the site's representative result for the brand query in clean/incognito sessions, 404-ing on click). It now meta-refreshes (0s, treated by Google as a permanent redirect) + canonicals to the homepage. Keep it OUT of `sitemap.xml`; do not delete until `site:tektsunami.com` no longer lists it.
8. **The About photo is INLINED in `elements.html`** (update#52) — a 640×960 WebP q75 (~81KB) embedded as a `data:` URI so it paints simultaneously with the text (zero network fetch; the earlier preload/`fetchpriority` approach still popped in ~0.5s late). `images/aboutme3.jpg` (640×960 q82) stays on disk ONLY because `index.html`'s JSON-LD `Person` block references it as `image` — deleting it would mean editing the JSON-LD and recomputing the CSP hash (see the CSP hard rule). To change the photo: regenerate the WebP (Pillow is now pip-installed; original 1000×1500 source is in git history pre-update#50) and re-embed.
9. **A branded OG share image exists: `images/og-share.png`** (1200×630, generated via .NET GDI+ — see update#27). If the brand/palette ever changes, regenerate it to match; it's referenced by `og:image`/`twitter:image` on all 11 content pages (`404.html` is `noindex` and intentionally has no OG/canonical tags).
10. **`oxipng` is installed** (winget, `Shssoichiro.Oxipng`) for lossless PNG recompression — use it on any new/replaced screenshot before committing (`oxipng -o 4 --strip safe <file>`).
11. **12 pages now, not 11** — `404.html` was added in update#28 (GitHub Pages serves it automatically on any 404). It carries `noindex, nofollow` and is intentionally excluded from `sitemap.xml`.
12. **`images/apple-touch-icon.png`** (180×180) exists for mobile home-screen icons — rasterized from `logo-tek-favicon.svg` via a browser canvas (no SVG rasterizer is installed locally; GDI+/System.Drawing cannot read SVG or WebP). Regenerate the same way if the favicon design changes.
13. **Latin fonts are SELF-HOSTED since update#50** — DM Sans / Source Sans 3 / JetBrains Mono live in `assets/fonts/` (10 variable-font woff2 files, latin + latin-ext, mirrored from fonts.gstatic.com), declared via `@font-face` at the top of `theme.css`, with the 3 latin files preloaded (`<link rel="preload" as="font" crossorigin>`) in every page's `<head>` — this killed the cold-load fallback-font flash (headline briefly oversized). Only the two Japanese footer-quote fonts (Klee One / Shippori Mincho) still load from Google Fonts (their CJK subsetting is ~120 slices per font — impractical to mirror). Do NOT re-add the latin families to the Google Fonts `<link>`.
14. **CSP `script-src` allows exactly one inline script**, via a `sha256-` hash — the JSON-LD Person block on `index.html`. If that block's content ever changes, the hash MUST be recomputed (`openssl dgst -sha256 -binary <extracted-content> | openssl base64`) or the page will silently fail to render the structured data (CSP blocks it, no visible error to the user, only a console CSP violation). **Corollary (so a future QA doesn't chase it):** this hash makes `index.html`'s CSP string *intentionally different* from the other 11 pages — only index carries the `'sha256-…'` in `script-src`. A check expecting all 12 CSP strings byte-identical will flag index as a false positive; that's by design (current hash: `sha256-c8F/04IzJjG8AuqL/Yu3IzZm9r8CJMF+c5V78ssBwy8=`, verified matching in update#42's final QA).

### Hero background (§7)
Backdrop is set to **Wave Contours** (`images/hero-contours.svg` + diagonal dark gradient). The **Aurora** alternative (off-centre blue/cyan glows) is kept as a comment block directly above the active `background:` rule — swap the two blocks to switch.

### Hero wave band (§7) — reworked in update#32
The band below the nav (`.hero-waves`) is now **5 inline SVG layers** (`hw5`…`hw1`, back→front), NOT 3. Each is a seamless multi-harmonic curve (generated so all wavelengths divide 1200 → the `translateX(-50%)` loop is seamless). Motion = **parallax drift + vertical roll**: layers drift at graduated speeds/alternating directions via `heroWaveA/B` (front tier, ±5px roll) and `heroWaveC/D` (back tier, ±3px roll); each keyframe set rolls up-and-down **twice per drift cycle** with the X kept perfectly linear so the loop stays seamless. The **front wave (`hw1`)** has a deliberately larger amplitude and is faster (`heroWaveA 9s`) so it's the lively "active" surface against the calmer back layers; it carries the `.hero-wave-crest` stroke (a hairline that slowly shimmers via `heroCrestShimmer`). **No `hero-hero-sheen`/light-sweep and no binary "bytes" — both were prototyped and scrapped this session.** All frozen under `prefers-reduced-motion`. Wave paths were generated by a throwaway Python script (sine-sum sampler) — there is no committed generator; regenerate similarly if the shapes need changing.
